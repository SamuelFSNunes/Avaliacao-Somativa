JavaScript e Lógica de Programação

  Em seguida, iniciamos o estudo da linguagem de programação JavaScript. Para praticar e aplicar o conhecimento adquirido, utilizamos essa linguagem em sites e realizamos exercícios de lógica. Exploramos conceitos como laços de repetição, tipos de dados, variáveis, condicionais, operadores lógicos, funções e arrays

  Realizamos a criação de um site que possui dois campos de entrada, um para código HTML e outro para código JavaScript. Ao clicar no botão "inicializar", o site compila esses códigos e os combina com uma estilização em CSS.

  Com o objetivo de aprimorar nossa habilidade lógica, recebemos uma lista de exercícios que deveriam ser resolvidos utilizando apenas elementos de programação. Em resposta a isso, desenvolvemos um código que nos permitiu solucionar cada exercício.

  Ao final da matéria aprendemos a importância de se otimizar um código e como aplica-lá através da programação estruturada. Com aplicações básicas o programa consegue executar em milisegundos, porém com programações mais complexas ele pode ter um "gap" de desempenho, por isso devemos sempre nos atentar ao desempenho do nosso código.

POO e Java

  Com ajuda de um curso proporcionado pela empresa ORACLE, demos inicio a POO (Programação Orientada a Objeto), sendo assim aprendemos conceito de classes, objetos, métodos, construtores, herança, abstração, encapsulamento, polimorfismo e tratamento de erros.

  Para implementar a programação orientada a objetos (POO), utilizamos a linguagem de programação JAVA. Durante o processo de aprendizado, familiarizamo-nos com a sintaxe da linguagem e exploramos o uso de arrays bidimensionais. Para consolidar o conhecimento, trabalhamos em exercícios lógicos que envolviam exibição dessas arrays, cálculos, identificação de diagonais primárias e secundárias, bem como manipulação de linhas e colunas.

  Os grandes projetos realizados foram: Inventário e ScreenProtector, onde em ambos utilizamos a POO para tornar o programa funcional. O inventário consistia em um código que permitisse o usuário adicionar, remover ou alterar produtos de um inventário.

  O ScreenProtector foi um projeto onde utilizamos a classe "Graphics" para pode desenhar formas geométricas ou imagens em uma nova janela, utilizamos conceitos da POO para definir hierarquias para as formas, bem como seus métodos.

Banco de Dados

Inicialmente, fomos apresentados aos conceitos de banco de dados, que podem ser divididos em dois tipos: relacionais e não relacionais. Diante disso, utilizamos o SGBD "PostegreSQL" como nossa ferramenta de gerenciamento.

Os bancos de dados relacionais são organizados por meio de modelos como o MER (Modelo de Entidade-Relacionamento) e o DER (Diagrama de Entidade-Relacionamento). Esses modelos nos ajudam a dividir os dados em entidades, atributos e relacionamentos, facilitando a interpretação e compreensão do banco de dados. Sabendo disso, fomos divididos em Squads e tivemos como desafio criar um modelo MER, DER e um banco de dados com base nas orientações recebidas.

Durante a criação do banco de dados, utilizamos comandos DDL (Data Definition Language) para criar, alterar e excluir tabelas e colunas. Além disso, utilizamos comandos DML (Data Manipulation Language) para popular essas tabelas e, por fim, utilizamos o DQL (Data Query Language) para realizar consultas nesse banco de dados.

Após a criação do banco de dados, aprendemos sobre as três formas de normalização e aplicamos esses conceitos na tabela TITANIC. Todo esse trabalho foi realizado em um ambiente real de banco de dados fornecido pelo SENAI, onde aprendemos a importância de evitar comandos como "DELETE sem WHERE", que podem causar consequências indesejadas no banco de dados.





Automação Industrial

Robótica e Automação Industrial

  Após algumas semanas trabalhando com linguagens e métodos de alto nível, passamos a explorar os sistemas embarcados e seus conceitos fundamentais, incluindo componentes de hardware como memória RAM, HD, SSD, processadores e fontes de energia, bem como microcontroladores, microprocessadores e o conceito de memória. Também discutimos os softwares de aplicação associados a esses sistemas.

  Fomos apresentados aos conceitos de robótica, tipos de rôbos industriais e suas ferramentas. Os principais conceitos abordados foram as partes de um rôbo, número de eixos, graus de liberdade, capacidade de carga, parâmetros para a movimentação e precisão. Além disso, tivemos uma aula prática onde observamos esse funcionamento ao vivo.

Ultrassônico

Tinkercad

  Com uma compreensão abrangente desses conceitos, avançamos para o estudo de uma nova linguagem de programação: o C++. Essa linguagem de baixo nível é amplamente utilizada no desenvolvimento de softwares e dispositivos embarcados. Aprendemos sua sintaxe e aplicamos nossos conhecimentos diretamente no TINKERCAD, uma plataforma online que nos permite simular automações industriais e aplicações com arduino.

  Ao utilizar essa plataforma, podemos criar e simular circuitos eletrônicos, desenvolver protótipos de produtos e explorar conceitos de automação industrial de maneira intuitiva. Essa plataforma oferece uma variedade de componentes eletrônicos virtuais, como microcontroladores, sensores e atuadores, que podem ser conectados e programados para criar projetos funcionais.

Projetos

  Foram realizados projetos nesta plataforma usando uma simulação do Arduino. O primeiro desafio consistiu em criar um código capaz de medir a distância utilizando um sensor ultrassônico, o que nos ajudou a compreender melhor os conceitos de entrada de dados e programação em C++.

  No segundo e último desafio, tivemos que construir um semáforo que simulava um cruzamento com faixa de pedestres. Nesse caso, sempre que um pedestre pressionasse o botão, a ação seguinte seria desligar o semáforo dos carros e acionar o semáforo para os pedestres.
